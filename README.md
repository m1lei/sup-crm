## О проекте

**Sup CRM** — это легкая и модульная CRM-система для управления сделками, задачами и активностью менеджеров.

##  Архитектура проекта
Проект реализован с использованием паттерна **Service Layer**. Это позволяет полностью отделить бизнес-логику от HTTP-слоя (контроллеров), делая код тестируемым и легким в поддержке.
Большинство Blade шаблонов реализованны по принципу **Smart Blade Components**.



## Структура проекта
```
app/
├── Actions/           # Отдельные методы
├── Http/
│   ├── Controllers/   # принимают запрос -> вызывают Сервис -> дают ответ
│   ├── Requests/      # Валидация входных данных
├── Models/            # Eloquent модели и связи
├── Policies/          # проверка прав доступа
├── Services/          # Логика получения, обработки, изменение в бд данных
└── View/Components/Widgets  # классы view компонентов(Smart Components),
```
## особенности проекта
resource/views/components/widgets с приставкой quick быстрые компоненты которые не имеют своих классов(анонимные компоненты)

Чистые контроллеры: Благодаря выносу логики в Сервисы и валидации в Requests, контроллеры содержат минимум кода.
